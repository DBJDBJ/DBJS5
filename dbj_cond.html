dbj.cond()

Initial dbj.cond() from DBJ.ORG, worked only on values that can be compared for equality, with native '===' operator.
New dbj.cond() allows users to compare values for equality with other values of any type or arrays of them. Order is "first found, first served". Syntax is the same as before: 
dbj.cond(a,b,c,...,d) ; 
	// returns c if a equals b, otherwise returns the last argument: d
There can be any number of pairs before last argument and after the first argument. New behaviour is that arguments can be of any type including objects, array, dates, etc. Values of different types are deemed not equal. if second value to be checked is an array all its members will ne checled until the equal is found. var x = dbj.cond(v1,v2,v3,v4,v5,v6) is shorthand for: 
var x = if ( equal(v1,v2) ) v3 ;
else if ( equal(v1,v4) ) v5 ;
else v6 ; // x may be v3, v5 or v6
Where v2 and v4 can be arrays. In which case their members will be checked.
equal(a,b) is called comparator. Comparator is the function which returns true if a and b are of the same type, a equals b or a equals any member of the array b. If b is an array.

Examples

dbj.cond( 1, 1, "A", 2, "B", "C") // returns "A"""
dbj.cond( 1, [3,2,1],"A", 2, "B", "C") // returns "A" , 1 is found first in [3,2,1]
Any types can be compared meaningfully. For example: 
dbj.cond( /./, /./, "A", /$/, "B", "C") // returns "A"
Function types are also compared and not called as functions 
dbj.cond( function() {return 1;}, function(){return 1}, "A", function(){return 1}, "B", "C") // returns "A"
This behavior allows for functions dispatching. Example: 
 function disptacher ( fx ) {
return dbj.cond( fx, f1, f2, f3, f4, f5 ) ;
// returns f2,f4 or f5 if neither f1 or f3 are equal to fx
}
Please see online qUnit Tests, for further examples

Internals

  • Comparator in essence defines the behaviour of dbj.cond()
  • Default inbuilt comparator works for all types.
  • First argument of the dbj.cond() is the left side in the comparison. Second value can be single value or array.
  • Default comparator allows for arrays of values to be compared with the single input value of the same type
  • Examples: 
    default_comparator( 1, [3,2,1] ) // --> true, 1 is found in [3,2,1]
default_comparator( function (){ return 1;}, [3,2,1] ) // --> false, input value is not executed as function
default_comparator( [3,2,1], [3,2,1] ) // --> true, comparing equal arrays
default_comparator( [3,2,1], [[7,8],[3,2,1]] ) // --> true, second array is equal to input
  • First argument od the comparator is also caled "input"
